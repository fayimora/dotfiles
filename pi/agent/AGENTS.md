# Development Guidelines

- Detect and follow the existing project's architecture, conventions, and tooling.
- Keep changes focused and maintain separation of concerns.
- Test meaningful behavior changes where practical.
- Update documentation when changing public APIs or configuration.
- Consider accessibility for user-facing interfaces.
- Measure before making performance optimizations.
- Use environment-based configuration; never commit secrets.
- Use structured logging and never log sensitive data.
- Follow the repository's existing database and Git conventions.
